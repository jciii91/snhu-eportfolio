Code Review

  • Pokémon Stadium Randomizer

    Download Original Artifact | Download Enhanced Artifact

    An application to randomize certain values in the game Pokémon Stadium for Nintendo 64. Users can upload their copy of the game to the program, and it will return a version of the game but with numbers shuffled around with the goal of providing a new experience. Originally this artifact was written in Java and was created and published in Fall 2023. I selected this item for my portfolio because I believe it is a unique project and provides a chance to showcase many different skills that I have developed over the course of my schoolwork and career.

    • Desktop Application to WebApp

      Originally this application was written in a Java and was written to be a desktop app. After releasing the application I received feedback from users that it was difficult to setup the environment to run the randomizer, or their computers would flag the application as a potential virus. I decided the best way to resolve these issues was to convert the randomizer to a webapp instead. I rewrote the randomizer in Python and constructed a frontend using Angular.

    • Capturing Settings through FormData and Dictionaries

      This second enhancement focused on expanding the capabilities of the Python portion of the webapp. Several of the functions used now accept levels of randomization, passed in as integers, that govern the limits of what the resulting in-game numbers can be. These settings are captured using FormData on the frontend and then recorded in dictionaries on the backend.

  • Travlr Getaways

    Download Original Artifact | Download Enhanced Artifact

    This project is a mock of a travel booking website. Visitors can view a list of available trips and admin users can add as well as edit existing trips. The original version of this artifact was submitted as a final project for my full stack development course and serves to showcase my knowledge of the MEAN stack.

    • Building a Dashboard

      In addition to the Trip Listings page, the website now offers a Dashboard for users to view trips that they have been booked for, or trips they have booked for other users. To facilitate this, I added Agent and Travler roles alongside the existing Admin role. A new collection, Bookings, was added to the database. This new collection ties together the travler, agent, and booked trip using their respectives identification codes. Results displayed on the dashboard are broken down by Traveler ID, and Agents can further filter the results to view only a single Traveler's bookings.
